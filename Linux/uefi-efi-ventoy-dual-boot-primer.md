# UEFI、EFI、Ventoy、ISO 与双系统原理笔记

**适用场景：** 理解笔记本如何从 U 盘启动、如何安装双系统、Ventoy 到底在做什么
**当前机器示例：** Lenovo ThinkBook 14 G5 IRH，1TB NVMe SSD，当前为 Ubuntu 24.04
**最后更新：** 2026-03-31

---

## 一、先记住一句最核心的话

双系统的本质就是：

**一块硬盘里分出多个区域，每个系统住在自己的区域里，开机时由启动程序决定先进入哪一个系统。**

---

## 二、几个容易混淆的角色

可以把整件事看成 6 个角色：

- `U 盘`：一块外接小磁盘，本质上和 SSD 一样，都是存数据的闪存设备
- `ISO 文件`：安装光盘的电子版，里面装着一个系统的安装内容
- `UEFI`：主板里的开机管理员，负责决定从哪里启动
- `EFI 分区`：磁盘上专门存放启动文件的小分区
- `Ventoy`：装在 U 盘里的启动工具，用来从一根 U 盘启动多个 ISO
- `Windows / Ubuntu`：真正的大系统，安装后住在各自的大分区里

关系图：

```text
按电源
  ->
主板里的 UEFI
  ->
决定从内置 SSD 还是 U 盘启动
  ->
去对应设备上的 EFI 启动区域找 .efi 文件
  ->
执行 .efi 文件
  ->
继续把 Windows 或 Ubuntu 真正启动起来
```

---

## 三、U 盘是什么，启动盘又是什么意思

### 1. U 盘本质上是什么

U 盘本质上就是一块小型闪存盘。

- 它可以被系统识别为一个磁盘
- 它可以分区、格式化、存文件
- 它也可以像硬盘一样被做成“可启动介质”

所以从启动角度看，U 盘就是一块外接磁盘。

### 2. 启动盘是什么意思

“启动盘”不是说系统长期装在 U 盘里，而是说：

**电脑开机时，先从这块 U 盘开始执行启动程序。**

正常开机：

```text
电源
-> UEFI
-> 内置 SSD 的启动文件
-> 进入系统
```

从 U 盘启动：

```text
电源
-> UEFI
-> U 盘里的启动文件
-> 进入安装环境或 Live 环境
```

---

## 四、ISO 文件是什么

ISO 可以理解成：

**一张安装光盘的完整电子镜像。**

例如：

- `Windows.iso`
- `ubuntu-24.04-desktop-amd64.iso`

ISO 里通常包含：

- 安装程序
- 启动文件
- 系统文件
- 恢复工具

但 ISO 本身只是一个文件，不是已经装好的系统。

### ISO 被启动，不等于系统已经装进 SSD

当你从 Ventoy 或普通启动盘里启动 `Windows.iso` 时，实际发生的是：

```text
U 盘把 Windows 安装环境启动起来
-> 你看到 Windows 安装界面
-> 这时 Windows 还主要在 U 盘和内存里
-> 只有你继续点安装，Windows 才会被写入 SSD
```

所以：

- “启动 ISO” = 进入安装环境
- “安装系统” = 把系统文件真正写进 SSD

---

## 五、UEFI 是什么

UEFI 是主板里的固件程序，可以理解成：

**电脑通电后最先醒来的底层启动管理器。**

它负责：

- 初始化硬件
- 识别硬盘和 U 盘
- 查看启动顺序
- 去磁盘上寻找可执行的启动文件

UEFI 不等于 Windows，不等于 Ubuntu，也不等于某个磁盘分区。

它在主板上，不在 SSD 或 U 盘里。

---

## 六、EFI 分区是什么

EFI 分区是磁盘上的一个小分区，通常几百 MB。

它的作用不是存放完整系统，而是：

**给 UEFI 提供可执行的启动文件。**

### 这句话怎么理解

UEFI 开机后不会直接把几百 GB 的 Windows 或 Ubuntu 全部理解并启动。
它会先去找一个它认识的小型启动程序，也就是 `.efi` 文件。

流程是：

```text
UEFI
-> 找到 EFI 分区
-> 找到某个 .efi 文件
-> 执行这个 .efi 文件
-> 由这个 .efi 文件继续把系统拉起来
```

所以 EFI 分区更像：

- 目录页
- 钥匙柜
- 门卫室

它不装系统本体，只装系统的“开机入口”。

### 为什么 EFI 分区很小

因为里面通常只放：

- `.efi` 启动程序
- 启动配置
- 少量辅助文件

真正体积很大的东西，例如：

- Windows 系统文件
- Ubuntu 系统文件
- 用户数据

都在各自的大分区里，不在 EFI 分区里。

---

## 七、Windows 和 Ubuntu 的启动文件是什么

EFI 分区里常见的启动文件大概长这样：

### Windows 常见路径

```text
\EFI\Microsoft\Boot\bootmgfw.efi
```

作用：

- 读取 Windows 启动配置
- 找到 Windows 系统分区
- 继续加载 Windows 的后续启动组件

### Ubuntu 常见路径

```text
\EFI\ubuntu\shimx64.efi
\EFI\ubuntu\grubx64.efi
```

作用：

- `shimx64.efi`：常用于 Secure Boot 场景下的过渡入口
- `grubx64.efi`：GRUB 启动器本体，负责显示菜单、找到 Linux 内核并继续启动 Ubuntu

### `.efi` 文件本质上是什么

可以粗略理解成：

- Windows 世界里有 `.exe`
- UEFI 世界里有 `.efi`

`.efi` 文件不是完整系统，而是给 UEFI 运行的启动程序。

---

## 八、Windows 和 Ubuntu 的启动链

### Windows 启动链

```text
电源
-> UEFI
-> EFI 分区中的 bootmgfw.efi
-> Windows Boot Manager
-> Windows 系统分区
-> Windows 内核
-> 进入 Windows
```

### Ubuntu 启动链

```text
电源
-> UEFI
-> EFI 分区中的 shimx64.efi
-> grubx64.efi
-> GRUB 菜单
-> Linux 内核 + initramfs
-> 进入 Ubuntu
```

这也是为什么“启动项坏了”和“系统本体坏了”是两回事。

- 启动项坏了：可能只是入口丢了
- 系统本体坏了：系统分区里的内容真的损坏了

---

## 九、Ventoy 是什么

Ventoy 不是操作系统，也不是 U 盘本身。

Ventoy 可以理解成：

**一个装在 U 盘里的多系统启动工具。**

它的作用是让一根 U 盘可以同时启动多个 ISO。

例如一根装了 Ventoy 的 U 盘里，可以直接放：

- `Windows.iso`
- `Ubuntu.iso`
- `PE.iso`
- `GParted.iso`

开机后，Ventoy 会先显示一个菜单，让你选要启动哪个 ISO。

### 普通 Windows 安装 U 盘和 Ventoy U 盘的区别

普通安装盘：

```text
U 盘
┌──────────────────────────────┐
│ 一整盘主要只服务一个安装环境 │
│ 例如只能装 Windows           │
└──────────────────────────────┘
```

Ventoy U 盘：

```text
U 盘
┌────────────┬──────────────────────────────────────┐
│ 启动区     │ 数据区                               │
│ 放 Ventoy  │ 直接存多个 ISO 文件                  │
└────────────┴──────────────────────────────────────┘
```

### Ventoy 的原理

流程可以理解成：

```text
电源
-> UEFI
-> 从 U 盘启动
-> 先运行 U 盘里的 Ventoy 启动程序
-> Ventoy 扫描 U 盘中的 ISO 文件
-> 你选择某个 ISO
-> Ventoy 再把这个 ISO 启动起来
```

所以 Ventoy 的本质是：

**它自己先被启动，然后它再替你启动你选中的 ISO。**

---

## 十、Ventoy 和 UEFI / EFI 的区别与联系

### 区别

- `UEFI`：主板里的固件程序
- `EFI 分区`：磁盘上的启动文件分区
- `Ventoy`：U 盘上的启动工具

三者不是同一个东西。

### 联系

Ventoy 并不是绕过 UEFI 工作，而是：

**利用 UEFI 的启动机制，让 U 盘先进入 Ventoy，再由 Ventoy 启动具体的 ISO。**

关系图：

```text
UEFI
-> 读取 U 盘上的 EFI 启动文件
-> 启动 Ventoy
-> Ventoy 启动 Windows.iso / Ubuntu.iso
```

同理，内置 SSD 上也有自己的 EFI 分区：

```text
UEFI
-> 读取 SSD 上 EFI 分区里的 .efi 文件
-> 启动 Ubuntu 或 Windows
```

所以：

- 内置 SSD 可以有自己的 EFI 分区
- U 盘也可以有自己的 EFI 启动区域
- UEFI 会根据你选的启动设备，去对应设备上找启动文件

---

## 十一、多系统是什么意思

“多系统”不一定只指安装到内置硬盘里的双系统，也可以指：

- 一根 U 盘里放多个安装镜像
- 一台电脑里装多个操作系统

### U 盘层面的多系统

意思是：

**同一根 U 盘可以启动多个不同的 ISO。**

这就是 Ventoy 最擅长的事。

### 内置硬盘层面的多系统

意思是：

**同一块 SSD 上安装多个操作系统，开机时二选一。**

例如：

- Ubuntu + Windows
- Ubuntu + Fedora
- Windows 11 + Windows 10

---

## 十二、双系统的磁盘布局原理

双系统不是“两个系统混在一起”，而是：

**每个系统住在自己的分区里，共用或协调使用启动分区。**

### 当前这台 ThinkBook 的磁盘布局

这台机器当前实际情况大致是：

```text
1TB NVMe SSD

┌──────────────┬────────────────────────────────────────────────────┐
│ EFI 分区     │ Ubuntu 分区                                       │
│ 512MB        │ 约 953GB，ext4，几乎占满整块盘                    │
└──────────────┴────────────────────────────────────────────────────┘
```

大白话解释：

- 小的 `EFI 分区` 是门卫室
- 大的 `Ubuntu 分区` 是当前唯一的大房间
- 现在还没有给 Windows 留出房间

### 未来装完双系统后的典型布局

如果给 Ubuntu 留约 `300GB`，给 Windows 留剩余空间，典型会变成：

```text
┌──────────────┬──────────────────────┬────────────┬──────────────┬────────────┐
│ EFI 分区     │ Ubuntu 分区          │ MSR/保留   │ Windows C 盘 │ 恢复分区   │
│ 512MB        │ 约 300GB             │ 约 16MB    │ 约 650GB     │ 几百 MB    │
└──────────────┴──────────────────────┴────────────┴──────────────┴────────────┘
```

其中：

- `EFI 分区`：继续保留，Windows 和 Ubuntu 都会用到
- `Ubuntu 分区`：缩小后保留给 Ubuntu
- `MSR/保留分区`：Windows 自动创建的小保留分区
- `Windows C 盘`：Windows 的系统分区
- `恢复分区`：Windows 的恢复工具分区

---

## 十三、为什么当前这台机器不能直接装 Windows

因为当前 SSD 上没有“未分配空间”。

现在的大分区几乎都给了 Ubuntu：

```text
[EFI][Ubuntu 超大分区]
```

而 Windows 需要的是：

```text
[EFI][Ubuntu 缩小后的分区][未分配空间]
```

然后安装 Windows 时，它才会把“未分配空间”变成自己的分区。

### 为什么不能在当前运行的 Ubuntu 里直接缩根分区

因为你现在正住在这个分区里。

大白话说：

- 人正在房间里住着
- 不适合直接在里面移动承重墙

所以通常要：

- 用 Ubuntu Live 或 GParted Live 从外部启动
- 在“系统没有被占用”的状态下缩小 Ubuntu 分区

这也是为什么装双系统时通常需要一个外部启动介质。

---

## 十四、一个容易记住的完整流程

安装 Ubuntu + Windows 双系统，通常是这个顺序：

```text
1. 用外部启动盘进入 Live 环境
2. 缩小现有 Ubuntu 分区
3. 产生未分配空间
4. 再从 Windows 安装盘启动
5. 把 Windows 安装到未分配空间
6. 最后检查启动项，必要时修复 GRUB
```

核心原则：

- 不删除现有 EFI 分区
- 不误删 Ubuntu 分区
- Windows 只装到未分配空间

## 十五、常见误区

### 误区 1：启动了 Windows ISO，就等于装好了 Windows

不是。

- 启动 ISO 只是进入安装环境
- 只有安装程序把文件写进 SSD，Windows 才算真正安装完成

### 误区 2：EFI 分区里装着整个 Windows 或 Ubuntu

不是。

- EFI 分区只放启动入口
- 真正的大系统在各自的大分区里

### 误区 3：装完 Windows 后进不了 Ubuntu，就说明 Ubuntu 被删了

不一定。

常见情况只是：

- Windows 改了默认启动项
- Ubuntu 还在，只是没有默认优先进入

### 误区 4：1TB SSD 在系统里显示约 953GB，说明少了几十 GB

不一定是少了。

通常只是：

- 厂商按 1000 进制标容量
- 系统按 1024 进制显示容量

所以：

```text
标称 1TB
-> 系统里常见约 953GiB
```

---

## 十六、最后用一句话重新压缩

如果只记一句话，可以记这个：

**UEFI 是主板里的启动管理者，EFI 分区是磁盘上的启动入口仓库，Ventoy 是 U 盘上的多 ISO 启动工具，ISO 是安装镜像，而双系统就是让多个系统分别住进 SSD 的不同分区里，由启动链决定开机先进哪一个。**
