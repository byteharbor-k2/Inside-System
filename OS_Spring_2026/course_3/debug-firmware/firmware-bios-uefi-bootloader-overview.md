# Firmware、BIOS、UEFI、Bootloader 总览

## 先给一个大图

很多人第一次接触这些词，会把它们都理解成“开机时电脑先运行的东西”。这个感觉不算错，但还不够细。

一个更完整的启动链，通常更像这样：

```text
CPU Reset
-> 最早期的启动代码
-> 固件 (Firmware)
-> Bootloader
-> 操作系统内核
-> 用户空间程序
```

不同平台的层次会有差别，但基本逻辑都差不多：

- 机器刚上电时，CPU 不知道什么叫 Windows、Linux、Android
- 它只会从规定的位置开始执行第一段代码
- 这段代码把硬件带起来，再把控制权交给后面的程序
- 最后才轮到操作系统内核和应用程序

所以这几个词并不是并列关系，而是同一条启动链上不同位置的概念。

## Firmware 是什么

`Firmware` 是最宽泛、最不容易错的总称。

可以把它理解成：

> 厂商预先放在机器里的底层代码，用来把硬件初始化好，并把控制权交给后续系统软件。

它的常见工作包括：

- 初始化 CPU 以外的硬件
- 建立最小运行环境
- 探测内存、存储、总线、外设
- 选择启动介质
- 校验下一阶段程序是否可信
- 把下一阶段程序加载到内存

大白话说，Firmware 干的是“开机后的杂活和准备工作”。

## ROM、Flash、Boot ROM 是什么关系

这里也很容易混。

- `ROM`
  早期真的是只读存储器，代码一旦写进去就很难改
- `Flash`
  现代很多设备的固件其实存放在可重写 Flash 里，所以可以升级固件
- `Boot ROM`
  指最早期、最底层的那一小段启动代码，很多时候真的是芯片内部固化好的

所以“固件写死在 ROM 中”这句话在历史语境里大致没问题，但放到现代设备上不够精确。

更准确的说法应该是：

- 有些最早期启动代码在芯片内部的 Boot ROM
- 更大、更复杂的固件常常放在板子上的 Flash 里

## BIOS 是什么

`BIOS` 全称是 `Basic Input/Output System`。

它是传统 PC 时代的固件和启动接口。很多老一代电脑人一说“电脑开机固件”，脑子里默认就是 BIOS。

它的几个关键特点是：

- 历史包袱很重，和 16 位实模式紧密相关
- 通过 BIOS 中断向系统程序提供基础服务
- 常见启动方式是读磁盘前 512 字节，也就是 MBR
- 再把控制权直接跳给这 512 字节里的代码

也就是说，传统 BIOS 的工作方式比较“朴素”：

- 先初始化硬件
- 再找启动盘
- 读一个扇区
- 最后跳过去执行

这就是为什么传统 PC 启动链里会经常看到：

```text
BIOS -> MBR -> Bootloader -> OS
```

## UEFI 是什么

`UEFI` 全称是 `Unified Extensible Firmware Interface`。

它可以理解成现代 PC 世界对 BIOS 的升级和替代。

和 BIOS 相比，UEFI 更像一个更完整、更现代的预启动环境。它不是只会“读 512 字节然后跳走”，而是能做更多事情，例如：

- 支持 32 位或 64 位预启动环境
- 能识别文件系统
- 能直接加载 `.efi` 程序
- 有更规范的启动项管理
- 更好支持 GPT 和大磁盘
- 支持 Secure Boot 等安全机制

如果用大白话区分：

- BIOS 更像“老式引导员”
- UEFI 更像“现代启动管理器 + 固件环境”

所以：

- BIOS 是固件的一种历史形态
- UEFI 也是固件的一种现代形态
- 它们都属于 Firmware 这个大类

## Bootloader 是什么

`Bootloader` 就是“启动加载器”。

它和 Firmware 不是一回事，但两者通常前后衔接。

Bootloader 的典型职责包括：

- 找到操作系统内核镜像
- 把内核和相关数据加载到内存
- 传递启动参数
- 准备内核需要的环境
- 最后跳转到内核入口

常见例子有：

- PC 上的 `GRUB`
- 嵌入式和开发板上常见的 `U-Boot`
- 手机上的各级 bootloader

可以这样理解：

- Firmware 负责把机器带到“能启动系统”的状态
- Bootloader 负责把“具体哪个系统、怎么启动”这件事处理掉

## 这几个概念之间到底怎么分层

最容易记住的方式是：

- `Firmware`
  最大的总称，表示机器自带的底层启动代码
- `BIOS`
  老式 PC 固件
- `UEFI`
  现代 PC 固件接口和环境
- `Bootloader`
  固件之后、内核之前的启动加载器

所以严格说：

- BIOS 和 UEFI 是 Firmware 的两种典型形式
- Bootloader 往往不是 Firmware 本身，而是下一阶段程序

不过在一些设备上，层次会继续拆开：

```text
Boot ROM -> 第一阶段固件 -> 第二阶段固件 -> Bootloader -> Kernel
```

这也是为什么你在 Android、RISC-V、ARM 开发板上，会看到名字比 PC 上更多、更碎。

## 为什么 BIOS 和 UEFI 经常被混用

今天互联网上说“进 BIOS”，很多时候其实是在说三件不同的事：

- 严格意义上的 `Legacy BIOS`
- 现代机器里的 `UEFI` 固件设置界面
- 泛指“开机按 F2/Delete 进去的那个固件菜单”

所以很多玩家、维修店、教程作者，甚至一些厂商文档，都会把“开机固件设置界面”统称成 BIOS。

这是口语习惯，不是严格定义。

可以这样区分：

- 技术语境里：
  - `BIOS` 是老式 PC 固件接口
  - `UEFI` 是现代 PC 固件接口
- 日常口语里：
  - “BIOS” 经常只是“开机设置界面”的代称

这就像很多人会把所有复印机都叫成一个老品牌名一样，是历史惯性。

## BIOS 和 UEFI 的演变历史

可以用一条简单时间线来理解：

- 1981 年，IBM PC 时代的 BIOS 成为 PC 兼容机的基础
- 之后很多年，AMI、Award、Phoenix 等 BIOS 厂商把这套生态做大
- 随着磁盘变大、硬件变复杂、启动需求变多，传统 BIOS 越来越不够用了
- Intel 后来提出 EFI 思路，想替代老 BIOS
- 再之后，UEFI Forum 推动 EFI 演变成更通用的 UEFI 标准
- 现代 PC 大多数已经以 UEFI 为主

在这个过渡期，很多主板还会提供：

- `Legacy Boot`
- `CSM`

这些兼容模式让 UEFI 机器还能模拟旧式 BIOS 启动方式。也正因为这样，普通用户更容易觉得“反正都叫 BIOS 就行了”。

## 不同平台上的表现不一样

### 1. 传统 x86 PC

更经典的链路是：

```text
Reset -> BIOS -> MBR -> Bootloader -> OS
```

现代一些机器则更常见：

```text
Reset -> UEFI -> EFI Boot Manager / GRUB -> OS
```

### 2. Android 手机

手机上你也会听到 bootloader，但很少有人会说 BIOS，因为手机不是 PC 这套历史。

更常见的理解是：

```text
Boot ROM -> 厂商固件/各级 bootloader -> Android 内核 -> Android 系统
```

这里常见的关键词是：

- Boot ROM
- Bootloader
- Verified Boot
- AVB
- vbmeta

### 3. ARM / RISC-V 开发板

常见会再拆得更细：

```text
Boot ROM -> 平台固件 -> OpenSBI / TF-A -> U-Boot -> Linux
```

所以一旦离开 PC 世界，`BIOS` 这个词通常就不再适用，而 `Firmware` 这个总称会更稳。

## 常见误解

### 误解 1：任何电脑都有 BIOS

不严谨。

更准确地说：

- 老式 PC 是 BIOS
- 现代大多数 PC 实际是 UEFI
- 只是很多人还沿用 BIOS 这个口语说法

### 误解 2：Firmware 就等于 BIOS

不对。

Firmware 是总称，BIOS 只是其中一种历史形态。

### 误解 3：Bootloader 就是固件

不一定。

Bootloader 常常是固件之后、内核之前的一段程序。虽然它也处在启动链里，但概念上最好和固件分开。

### 误解 4：固件总是写死在 ROM，不能改

这是老印象。

现代很多固件都可以升级，只是最早期那一小段 Boot ROM 往往不可改。

## 一句话总结

如果你只记一句话，可以记这个：

> Firmware 是“机器自带的启动底层代码”的总称；BIOS 是传统 PC 固件，UEFI 是现代 PC 固件环境，Bootloader 是把操作系统内核真正装起来的下一阶段程序。

再换成更口语一点的话：

- Firmware 负责“把机器叫醒”
- BIOS/UEFI 是 PC 世界里两代不同的“叫醒方式”
- Bootloader 负责“把操作系统领进门”
