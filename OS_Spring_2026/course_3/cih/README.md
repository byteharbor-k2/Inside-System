# CIH 病毒

CIH (也称为 Chernobyl 病毒) 是台湾大同大学学生陈盈豪于 1998 年创建的，能够影响早期的 Windows 9x (95、98、Me) 操作系统。在 AI 的帮助下，我们把病毒代码翻译成了 “好读” 的 C 语言，可以清晰地看到病毒感染、潜伏和破坏的过程。


1. 提权进入 Ring 0
    在 cih.c:48 和 cih-1.4.asm:178 里，病毒先借助 Win9x 对保护机制很弱的特点，读取并改写 IDT 中某个异常向量，然后主动触发中断，让 CPU 以 Ring 0 权限执行它
    自己的处理代码。
    这一步是 CIH 的核心前提，因为后面感染文件、改 BIOS、改磁盘都需要系统级权限。
2. 驻留内存并挂文件系统钩子
    在 cih.c:86 和 cih-1.4.asm:313 里，进入 Ring 0 后它会分配内核内存，把自己常驻进去，并安装 IFSMgr_InstallFileSystemApiHook 的文件系统钩子。
    这样以后系统每次打开文件，它都能在底层看到请求，做到“打开即感染”。它还用 DR0 一类寄存器当“是否已驻留”的标记，避免重复装载。
3. 用“孔洞感染”感染 EXE，而不增大文件体积
    这是 CIH 最有名的地方，逻辑在 cih.c:116 和 cih-1.4.asm:455。
    它只盯 .EXE，读取 PE 头，检查是否已感染，然后计算各节区 SizeOfRawData - VirtualSize 的空隙，把病毒代码拆碎塞进这些对齐留下的 slack space 里，再把入口点
    改到病毒第一段。这样文件大小不变，更难被当年的杀毒软件靠“体积变化”发现。
    运行时它再把这些碎片重新拼回完整病毒体，所以你会在 cih-1.4.asm:235 看到“merge all virus code section”的逻辑。它还会恢复只读属性、恢复修改时间，尽量降低
    痕迹。
4. 到触发日期后直接破坏 BIOS 和硬盘
    破坏逻辑在 cih.c:212 和 cih-1.4.asm:1206。
    它不是通过操作系统 API，而是直接读 CMOS 判断日期是否为 26 号；满足条件后，尝试解开 EEPROM/Flash 写保护并覆盖 BIOS 映射区，同时调用底层磁盘 I/O 去覆盖物
    理扇区。
    所以它危险的地方不是“会感染固件”，而是“会在特定日期直接破坏固件和磁盘内容”。
